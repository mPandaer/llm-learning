# LLM 的基本认知

探索技术的本质，享受技术的乐趣！大家好，我是长歌，今天我想给大家介绍一些我们应该对 LLM 有的基本认知，大概内容如下：

1. LLM 的基本原理是什么？
2. LLM 有哪些能力？
3. LLM 与传统程序交互的时候，应该注意什么？

## LLM的基本原理

从现象上看，大语言模型只在干一件事情：**不知疲倦的生成文本**，只不过生成的文本比较有逻辑性，连贯性，以至于让人们误以为 LLM 具备了逻辑，从而认为 LLM 具备了智能。但是 LLM 在生成文本的过程中，根本不会向人类一样思考文字背后的逻辑，只不过是**基于概率**的不断生成罢了。

<img src="./images/%E6%88%AA%E5%B1%8F2026-04-25%2011.17.08.png" alt="截屏2026-04-25 11.17.08" style="zoom:50%;" />

当我们输入几个字：“今天天气”给大模型之后，大模型在内部通过运算，计算出生成下一个字的概率，从概率比较高的字中，根据一定的规则取出一个字，拼接到原始文本的后面，即“今天天气不错”，然后将“今天天气不错”又当做原始文本交给大模型，继续生成下一个字，就这样周而复始。我们就看见了大模型生成了一段比较完整的句子了。

这就是大模型的底层原理，从整体来看，就是这么朴素简单，因此，我希望你理解大模型的本质后，能够对大模型祛魅，大模型还没有具备智能，只不过因为它能够根据**概率**不断的生成文本且生成的文本符合逻辑，让它看起来具备所谓的“智能”罢了。



## LLM的两大核心能力

虽然大模型不具备智能，但是不代表它能力不行，整体上来看，他拥有两大核心能力：

1. 自然语言理解能力 （PS：自然语言就是人与人之间沟通的语言）
2. 不错的推理能力

拥有自然语言理解能力，就可以改变应用与人之间的交互方式，人与应用之间通过自然语言交互，再也不需要填写表单呀，用鼠标点点呀。

拥有了推理能力，就可以把一部分传统程序解决的不够好的问题，让大模型来解决。比如在复杂的场景，如何决策下一步该怎么办的问题？

当我们将大模型的两大能力集成到我们的应用中的时候，究竟哪些问题是大模型该解决的？哪些问题是传统程序该解决的？我的判断如下：

1. 只要涉及到自然语言理解的，就只能交给大模型
2. 只要是确定性的，交给传统程序解决，性价比更高，而且更准确和稳定。

于是，在我看来，**大模型是全能的，但是它不稳定，慢。传统程序不是全能的，只能解决确定性的问题，但是它稳定，准确，快**。



## LLM与传统程序交互的注意事项

当我们将 LLM 集成到我们的传统程序中时，我们不得不考虑一个比较现实的问题，LLM 与传统程序之间，数据应该通过什么样的方式来进行交互呢？我们先看看传统程序和 LLM 都支持什么样的数据来交互：

**传统程序**：必须是结构化的数据，而且是需要定义好的结构

**LLM**：因为我拥有自然理解能力，所以我既可以是自然语言，也可以是结构化的数据

通过以上的对比，就知道了，传统程序与 LLM 进行数据交换的时候，只能是结构化的数据。但是有一类对话式的场景，即需要给用户输出自然语言，又需要用结构化的数据渲染前端，而且这个结构化的数据还只能由大模型从用户的对话中提取。在这样的场景下，会出现一个矛盾，自然语言数据天然适合流式返回，而结构化数据天然不适合流式返回。为了缓解这个矛盾，一般有两种做法：

1. 让 LLM 即输出自然语言，又输出结构化的文本，通过一定的分隔符进行分隔。 缺点：稳定性不好，解析流式文本的程序难度大
2. 先让 LLM 生成结构化数据，再让 LLM 基于这个结构化数据，做出自然语言解释。缺点：因为多了一步自然语言解释，会慢一些。
