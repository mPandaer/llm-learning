# 一个想法
我想实践一下费曼学习法，我的想法大概是这样的，我打算制作一门开源教程，主要负责教会有一定编程基础的人员，学习 AI 应用开发，以及大模型相关的算法以及对应的落地实践，比如模型微调，算法复现等等。所以这个教程我分为了两个阶段，第一个阶段就是AI 应用开发，第二个阶段就是大模型相关的原理与算法。
以上就是我的想法，但是我的现状：我只是一个刚刚毕业一年的Java 后端开发人员，我想利用我自己写这个教程，通过输出来学习相关的 AI 应用开发技术，以及大模型相关的原理。
这是整个教程的内容，教程内容的呈现形式是：文档 + 代码库。代码库肯定就放在 GitHub 上，具体点就是代码相关的东西放在projects目录中，文档（文字 + 图文）放在docs目录中。但是我希望文档能够以静态站点的方式呈现出来，比如借助 GitHub Pages。
现在我想把文档变成静态站点，这个基础的功能给搭建出来。这样后面只需要专心写文档就可以了。


# 站点目录结构的优化
我发现左侧栏对应的文章是在docs/.vitepress/config.ts中通过配置得到的，这不够纯粹，后续的工作流应该是我写一篇文章，然后就能自动生成这部分的配置，唯一需要确定的就是文章之间的顺序，我认为可以从文件名入手，比如以 001-大模型基础认知.md 002-提示词工程.md 只要按照这个规范来就可以知道文章的顺序了，如果没有前面这三个数字，就默认最后。
这个文章命名规则只在docs/项目实践 docs/第一阶段：AI 应用开发 docs/第二阶段：大模型原理与算法 中有效。
